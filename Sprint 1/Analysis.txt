Hotel Booking System – CozyHeven


1 Customer / User

The user registers an account by providing name, email, phone number, address, and password.

After successful registration, the user logs into the system.

The user searches for hotels based on location, price, rating, and available rooms.

The system displays a list of hotels along with ratings, description, amenities, and room details.

The user selects a hotel and views available room types, bed type, capacity, and price.

The user selects a room and enters check-in and check-out dates.

The user enters the number of adults and children and provides guest details.

The system calculates the total price based on room price and duration of stay.

The user confirms the booking.

The user proceeds to payment using UPI, card, or other payment methods.

Once payment is successful, the system confirms the booking and stores it in the database.

After completing the stay, the user can write a review and give a rating for the hotel.


Task:
Allow customers to easily search, book rooms, make payments, and review hotels.

2. Hotel Owner 


The hotel owner registers in the system and logs in.

The owner adds a new hotel by providing:

hotel name

location

description

ratings (optional initially)

The owner adds rooms under the hotel, specifying:

room size

bed type

maximum number of people

base price

AC type

availability status

The owner assigns amenities to the hotel such as:

WiFi

Parking

Swimming pool

Restaurant

The owner monitors room availability and bookings made by users.

When a booking is confirmed, the system updates the room availability automatically.

The owner can view customer reviews and ratings to improve service quality.

Task:
Enable hotel owners to manage hotels, rooms, amenities, and monitor bookings.


3.Payment System Story

After confirming booking details, the user proceeds to the payment stage.

The system calculates the total booking amount based on room price and stay duration.

Any discount or offer is applied if available.

The user selects a payment method (UPI, card, net banking, etc.).

The payment is processed through the payment gateway.

The system records:

payment amount
payment method
payment status
payment date

If the payment is successful, the booking status becomes confirmed.

If payment fails, the booking remains pending or cancelled.

Task
Securely process and store payment transactions.


4. Review System 

After completing the stay, the user logs into the system.

The user navigates to their booking history.

The user selects a completed booking.

The user submits a rating (1–5 stars).

The user writes a title and comment describing their experience.

The system stores the review with:

user ID

hotel ID

booking ID

stay date

Other users can see these reviews while searching for hotels.

The system calculates average hotel ratings based on reviews.

Task:
Help users choose better hotels through honest feedback.

5.Amenities System 

The system maintains a list of amenities such as WiFi, parking, gym, swimming pool, restaurant, etc.

When a hotel is created, the owner selects the amenities available in that hotel.

These amenities are stored in the Hotel_Amenities mapping table.

When users search for hotels, the system displays available amenities.

Users can filter hotels based on specific amenities.

Task:
Allow users to choose hotels based on available facilities.

6.Guest Management Story

During booking, the user specifies the number of adults and children.

The system ensures the number of guests does not exceed room capacity.

The user enters guest details such as name and age.

Each guest is stored in the Booking_Guests table linked to the booking.

This helps the hotel maintain accurate guest records.

Task:
Track the exact number of people staying in each booking.
